Многострочное расположение flex блоков

Как вы уже знаете, флекс-блоки по умолчанию не переносятся на новую строку, даже если не помещаются в родителя. Часто, однако, нам как раз-таки и нужен такой перенос.

Добиться переноса блоков на новую строку можно, указав свойство flex-wrap в значении wrap. В этом случае, все блоки, которые не поместились в ширину родителя, будут не сжиматься, как это было у нас ранее, а переносится на новую строку.

Пример

Давайте посмотрим на примере. Пусть у нас есть 4 блока с шириной 100px и родитель с шириной 250px. Без использования flex-wrap блоки-потомки уменьшат свою ширину так, чтобы поместиться в родителя:

<div class="parent"> <div class="child">1</div> <div class="child">2</div> <div class="child">3</div> <div class="child">4</div> </div> .parent { display: flex; flex-direction: row; width: 250px; border: 1px solid red; } .child { width: 100px; height: 100px; border: 1px solid green; }

:

Пример

Давайте теперь зададим свойство flex-wrap. В этом случае ширина блоков не будет уменьшаться, а не влезающие блоки будут переносится на новую строку.

Очевидно, что если ширина блоков не будет уменьшаться, то в наш родитель с шириной 250px поместятся только 2 блока в ряд и после каждого второго блока будет еще свободное пространство в 50px, вот так:

<div class="parent"> <div class="child">1</div> <div class="child">2</div> <div class="child">3</div> <div class="child">4</div> </div> .parent { display: flex; flex-direction: row; flex-wrap: wrap; /* разрешаем блокам переносится */ width: 250px; border: 1px solid red; } .child { width: 100px; height: 100px; border: 1px solid green; }

:

Пример

Давайте теперь зададим родителю ширину 200px, чтобы блоки-потомки помещались без отступа справа:

.parent { display: flex; flex-direction: row; flex-wrap: wrap; width: 200px; /* установим родителю ширину вместо 250px */ border: 1px solid red; }

В результате нас ждет сюрприз - блоки не будут помещаться по два в ряд:

Почему так произошло? Все дело в том, что реальная ширина блоков не 300px, а 302px - ведь границы расширяют наши элементы.

Почему нам это никогда не мешало ранее? Потому что раньше мы использовали однострочный вариант, и, если из-за границ блоки не помещались в родителя, то они просто немного сжимались на ширину границы.

Когда же мы написали flex-wrap, сжатие флекс-элементов прекратилось, их ширина стала такой, как написано в свойстве width, да еще плюс две границы, то есть 102px. Так как ширина родителя 200px, то 2 блока по 102px не могут стать в ряд - им просто не хватит места.

Для исправления проблемы добавим свойство box-sizing в значении border-box нашим блокам-потомкам. В этом случае границы перестанут раздвигать блоки и их ширина реально будет 300px и все будет работать, как надо:

.child { box-sizing: border-box; width: 100px; height: 100px; border: 1px solid green; }

Вот теперь все будет работать так, как надо:
